# 澄清问题库

## 提问策略

### 何时提问

- 信息缺失且对故事至关重要
- 存在多种有效解释
- 范围可能因答案显著变化
- 安全或合规影响不明确

### 何时假设（并记录）

- 标准行业惯例适用
- 系统中存在类似功能（引用它）
- 低影响决策，可轻松更改
- 用户明确表示"自行判断"

### 批量提问原则

- 将相关问题组合在一起
- 每次交互限制 3-5 个问题
- 按对故事结构的影响排序

## 问题分类

### 1. 用户/角色问题

- 这个功能的主要用户是谁？
- 是否有不同用户类型有不同需求？
- 用户的技术水平如何？
- 这是给内部用户、外部客户，还是两者都有？

### 2. 范围问题

- 最小可行版本包含什么？
- 有哪些功能可以延后到下一阶段？
- 需要支持移动端/平板/桌面吗？
- 有特定的浏览器/平台需要支持吗？

### 3. 业务逻辑问题

- 当 [边缘情况] 发生时怎么处理？
- 有任何限制（数量、大小、频率）吗？
- [输入] 的验证规则是什么？
- 错误应该如何向用户展示？

### 4. 集成问题

- 需要与现有功能连接吗？
- 涉及第三方服务吗？
- 需要与其他系统共享什么数据？
- 有 API 要求吗？

### 5. 数据问题

- 哪些数据是必填的，哪些是可选的？
- 有数据格式要求吗？
- 数据需要保留多长时间？
- 有隐私/合规考虑吗？

### 6. 非功能问题

- 有性能要求吗？
- 预期的用户负载是多少？
- 有超出标准的安全要求吗？
- 需要离线支持吗？

## 问题模板

### 模糊功能

```
您提到需要 "[feature]"，我需要了解:

1. **主要用户是谁?**
   - A: [选项1]
   - B: [选项2]

2. **这个功能的核心目标是什么?** (解决什么问题)

3. **有没有现有系统或竞品可以参考?**
```

### 范围不明确

```
关于 "[feature]" 的范围，请确认:

1. **MVP 版本必须包含哪些功能?**
   - 如不指定，我会假设: [默认]

2. **哪些可以作为后续迭代?**

3. **有没有明确不需要的功能?**
```

### 缺少边缘情况

```
对于 "[scenario]"，需要确定:

1. **当 [condition] 时应该如何处理?**
   - A: [选项1]
   - B: [选项2]

2. **是否有数量/大小限制?**
   - 如不指定，我会假设: 无限制

3. **出错时用户应该看到什么?**
```

### 用户权限

```
关于用户权限，请确认:

1. **哪些用户可以执行此操作?**
   - A: 所有用户
   - B: 仅登录用户
   - C: 特定角色

2. **未授权用户尝试访问时如何处理?**
```

## 按领域的常见问题

### 电商

- 是否支持游客结账？
- 需要多少种支付方式？
- 运费/税费计算规则？
- 库存管理方式？

### SaaS/B2B

- 定价层级和限制？
- 多租户要求？
- SSO/企业认证需求？
- 数据隔离要求？

### 移动应用

- 需要离线支持吗？
- 支持哪些平台（iOS/Android）？
- 推送通知需求？
- 后台处理需求？

### 内部系统

- 与现有系统的集成？
- 审批流程需求？
- 审计追踪要求？
- 数据导出需求？

## 提问优先级矩阵

| 信息类型 | 优先级 | 说明 |
|---------|--------|------|
| 用户角色 | 高 | 影响整个故事主语 |
| 核心功能 | 高 | 无法假设 |
| 边界范围 | 中 | 影响验收标准 |
| 错误处理 | 中 | 可参考通用模式 |
| 性能要求 | 低 | 可后续补充 |
| 技术细节 | 低 | 通常不需要问 |
